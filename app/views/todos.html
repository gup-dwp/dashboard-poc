Listing to-dos

Standard pattern

Use this when:

  • a list of things need to be done
  • each thing is assigned to a single user
  • each thing links to thing that needs to be done
  • completed things are stored and viewed elsewhere

Research notes

People found the layout easy to use and interact with, especially on mobile. It is important you use verbs for the to-do label so people clearly know what to expect. Whilst people positively responded to doing the to-dos in any order, most naturally completed them top to bottom, so it's important to order them logically for your service. If a to-do can be part-completed, an indication of progress is a must.

Alternative pattern

Use this when:

  • a list of things need to be done in a single view by multiple users
  • each thing is assigned to one or multiple users
  • each thing needs to have supporting content
  • each thing needs to be completed by a specific date
  • completed things are stored and viewed elsewhere
